# AI-Driven Role-Specific Interview System

A comprehensive Django web application that uses Google's Gemini AI to conduct role-specific technical interviews with automated question generation and evaluation.

## Features

### For Candidates
- **Secure Registration & Login**: Role-based authentication system
- **Resume Upload & Parsing**: Automatic extraction of name, email, and phone from PDF/DOCX files
- **Role Selection**: Choose from Frontend Developer, Backend Developer, or Data Analyst
- **AI-Powered Interviews**: 6 questions per interview (2 easy, 2 medium, 2 hard) generated by Gemini AI
- **Timed Responses**: 5-minute timer per question to simulate real interview conditions
- **Instant Feedback**: AI evaluation and scoring for each answer
- **Session Persistence**: Resume incomplete interviews
- **Progress Tracking**: Visual progress bars and completion status

### For Interviewers
- **Dashboard Overview**: View all candidates and their interview results
- **Search & Filter**: Filter candidates by name, role, or status
- **Detailed Reports**: Comprehensive interview reports with:
  - All questions and answers
  - Individual scores and AI feedback
  - Final holistic scores
  - AI-generated summaries
- **Resume Access**: Direct links to candidate resumes

### Technical Features
- **Google Gemini AI Integration**: Advanced AI for question generation and evaluation
- **Secure File Uploads**: Restricted to PDF and DOCX formats with size limits
- **Responsive Design**: Bootstrap-based UI that works on all devices
- **Real-time Updates**: AJAX-powered answer submission and evaluation
- **Role-based Access Control**: Separate dashboards and permissions
- **Database Persistence**: SQLite with proper relationships and constraints

## Tech Stack

- **Backend**: Python 3, Django 5.2
- **AI Integration**: Google Gemini API (gemini-pro)
- **Frontend**: HTML5, CSS3, JavaScript (Bootstrap 5)
- **File Processing**: PyPDF2, docx2txt
- **Database**: SQLite (development) / PostgreSQL (production)
- **Deployment**: Ready for Heroku, AWS, or any Django-compatible hosting

## Installation & Setup

### Prerequisites
- Python 3.8+
- pip package manager
- Google Gemini API key

### 1. Clone & Setup
```bash
# Navigate to the project directory
cd interview_project/

# Install dependencies (note: requirements.txt is in parent directory)
pip install -r ../requirements.txt
```

### 2. Environment Variables
Create a `.env` file in the project root (`interview_project/.env`):
```
GEMINI_API_KEY=your_gemini_api_key_here
```

Or set the environment variable:
```bash
export GEMINI_API_KEY="your_api_key_here"
```

**Note:** The `.env` file is already created for you. Replace `your_gemini_api_key_here` with your actual Gemini API key.

### 3. Database Setup
```bash
python manage.py migrate
```

### 4. Create Superuser (Optional)
```bash
python manage.py createsuperuser
```

### 5. Run Development Server
```bash
python manage.py runserver
```

Visit `http://127.0.0.1:8000/` to access the application.

## Usage

### For Candidates:
1. Register as a candidate
2. Upload your resume (PDF or DOCX)
3. Select your desired role
4. Answer AI-generated questions within time limits
5. View your scores and feedback
6. Resume incomplete interviews

### For Interviewers:
1. Register as an interviewer
2. View candidate dashboard with search/filter options
3. Click on candidates to view detailed interview reports
4. Access candidate resumes and AI evaluations

## API Endpoints

- `POST /api/submit-answer/`: Submit and evaluate interview answers (AJAX)
- All other interactions are handled through Django views

## Security Features

- **CSRF Protection**: Django's built-in CSRF middleware
- **XSS Protection**: Content type sniffing and XSS filters
- **File Upload Security**: Restricted file types and size limits
- **Role-based Access**: Separate permissions for candidates and interviewers
- **Session Security**: Secure session handling
- **Input Validation**: Form validation and sanitization

## Development

### Adding New Roles
1. Update `ROLE_CHOICES` in models.py
2. Add fallback questions in `gemini_integration.py`
3. Update role selection forms

### Customizing Questions
Modify the prompt in `GeminiInterviewService.generate_interview_questions()` to change question generation behavior.

### Styling
The application uses Bootstrap 5. Custom styles are in the base template. Static files are served from `core/static/core/`.
